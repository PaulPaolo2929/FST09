🧩 STEP 2 — USING TINKER TO ADD DATA
--------------------------------------------------
(Inserting records into your “students” table easily using Laravel’s built-in tool)

🧠 What is Tinker?
Tinker is like a Laravel sandbox — a place in your terminal where you can directly talk to your Laravel app (and your database) using PHP commands.
✅ You can:
Create new records
View data
Update or delete records
without making a web page yet.

----------------------------------------------------------
🖥️ Step-by-Step Guide
1️⃣ Open your terminal / CMD

Make sure you are inside your Laravel project folder:

cd STUDYLARAVEL

-------------------------------
2️⃣ Run Tinker

Type:
php artisan tinker
If it works, you’ll see something like:

Psy Shell v0.11.1 (PHP 8.x — cli) by Justin Hileman
>>> 

That means you’re now inside the Tinker shell (like Laravel’s mini command line).

-----------------------------------------------------------
3️⃣ Tell Laravel you want to use your Student model

Type this inside Tinker:

use App\Models\Student;

This makes the Student model ready to use — it represents your students table.

-------------------------------------------------------
4️⃣ Create a new student record

Now, type this:

Student::create([
    'name' => 'Juan Dela Cruz',
    'email' => 'juan@gmail.com'
]);

✅ What this does:
Inserts a new row into your students table
The name is Juan Dela Cruz
The email is juan@gmail.com

If it works, you’ll see something like:

=> App\Models\Student {#xxxx
     name: "Juan Dela Cruz",
     email: "juan@gmail.com",
     updated_at: "2025-10-05 08:00:00",
     created_at: "2025-10-05 08:00:00",
     id: 1,
   }

That means your new record has been added successfully. 🎉

---------------------------------------------------------
5️⃣ Check your database

Go to your browser → open:

http://localhost/phpmyadmin


Then click your database name (ex: studylanaveldb)
→ click the students table
→ and you’ll see your new record!

------------------------------------------------
6️⃣ Exit Tinker

When you’re done, just type:

exit

-------------------------------------------
💡 (Extra Tip)

If you get an error like:
SQLSTATE[HY000]: General error: 1364 Field 'name' doesn't have a default value

That means your model isn’t allowing “mass assignment” yet.
✅ To fix that:

Open your model file:

app/Models/Student.php


Add this inside the class:
protected $fillable = ['name', 'email'];

Now you can insert data again safely.

---------------------------------------------------------------------------------------
---------------------------------------------------------------------------------------
Error Handling:
If the Data inserted is not showing on the Phpmyadmin possible error:

Step 1: Check Your .env File (Database Settings)
Go to your Laravel project folder and open the file named:

.env

Inside, you’ll see something like this 👇
DB_CONNECTION=sqlite
DB_DATABASE=/absolute/path/to/database/database.sqlite

That means Laravel is using SQLite, not MySQL — which explains why phpMyAdmin (MySQL) shows no tables.

---------------------------------------
✅ Step 2: Change It to Use MySQL

Update your .env file to look like this:

DB_CONNECTION=mysql //usually this is default "sqlite" so change it
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=root
DB_PASSWORD=

⚠️ Replace:

your_database_name → put the name of your database from phpMyAdmin (e.g. laravel_db or student_db)
DB_USERNAME=root → usually correct for XAMPP users
DB_PASSWORD= → leave blank if your XAMPP MySQL has no password (default)

--------------------------------------------------------------------
🚀 Step 4: Migrate Again
After saving .env, go back to CMD and run:

php artisan migrate:fresh

✅ You should now see:

Migrating: 2025_10_12_000000_create_students_table
Migrated:  2025_10_12_000000_create_students_table (XX.XXms)

---------------------------------------------------------------------------
🧩 Step 5: Check phpMyAdmin
Go back to phpMyAdmin → open your database (e.g. laravel_db) → you should now see:

migrations
password_reset_tokens
personal_access_tokens
students
users


🎉 That means Laravel is now connected to MySQL and created all the tables.

----------------------------------------------------------------------
-----------------------------------------------------------------------
🧠 Bonus Tip:
If you still don’t see any tables, try:

php artisan config:clear
php artisan migrate

This forces Laravel to reload your .env file configuration.
