📘 **STEP 4: CONTROLLERS (Laravel Notes)**

🧠 What Is a Controller?
Controllers are the “brains” of your Laravel app.
They handle logic, get data, and return what the user should see.

💡 Think of it like this:
- Route = Door  
- Controller = Brain  
- View = Face  

---

⚙️ Create a Controller:

```bash
php artisan make:controller PageController
```

📁 File created at:
`app/Http/Controllers/PageController.php`

---

📄 Example Inside the Controller:

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PageController extends Controller
{
    public function home() {
        return view('home');
    }

    public function about() {
        return view('about');
    }
}
```

🧩 Each function handles one page or logic.

---

🔗 Connect Controller to Routes:

```php
use App\Http\Controllers\PageController;

Route::get('/', [PageController::class, 'home']);
Route::get('/about', [PageController::class, 'about']);
```

➡️ `/` shows `home.blade.php`  
➡️ `/about` shows `about.blade.php`

---

📁 Views Folder Example:

```
resources/
 └── views/
      ├── home.blade.php
      └── about.blade.php
```

---

✅ Why Use Controllers?
- Organizes logic and routes  
- Keeps code clean and reusable  
- Easier for large projects  

---

🧾 Bonus Tip: Create a Resource Controller (CRUD-ready)

```bash
php artisan make:controller StudentController --resource
```

This includes methods:
- index() → show all data  
- create() → form for new data  
- store() → save new data  
- edit() → edit form  
- update() → update record  
- destroy() → delete record
